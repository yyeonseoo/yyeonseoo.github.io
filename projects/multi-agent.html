Multi-Agent 프로젝트 : 정신 건강 상담 도메인을 주제로

정신 건강 상담 도메인에서 Fine-Tuning 없이 Multi-Agent 구조만으로 상담 품질을 향상시킬 수 있는지 단계별 실험과 분석을 통해 검증한 연구 프로젝트입니다.

01 · CONTEXT

배경과
문제 정의

최근 LLM 기반 정신 건강 상담 서비스가 증가하고 있지만, 상담 도메인은 잘못된 응답이 실제 사용자 안전에 영향을 미칠 수 있는 High-Risk Domain입니다.

기존 연구는 대부분 최종 응답 성능만 비교하기 때문에 어떤 Agent가 성능 향상에 기여했는지 설명하기 어려웠습니다. 이에 Fine-Tuning 없이 Agent 구조 설계만으로 상담 품질을 향상시킬 수 있는지를 연구 질문으로 설정하고, 성능뿐 아니라 구조의 타당성을 설명할 수 있는 평가 방법을 제안하고자 했습니다.

02 · OBJECTIVE

목표와
성공 기준

연구 목표

  • Multi-Agent 구조와 Single LLM의 상담 품질 비교
  • Agent 간 상호작용이 상담 품질에 미치는 영향 분석
  • 각 Agent의 역할과 기여도를 설명 가능한 형태로 검증

성공 기준

  • Safety, Faithfulness, Relevancy, Empathy 향상
  • 성능 향상 원인을 단계별 분석으로 설명
  • LLM as a Judge 기반의 일관된 평가 체계 구축

04 · APPROACH

핵심 접근
과정

기존 연구처럼 최종 응답만 비교하는 대신 Stage-wise Evaluation을 적용했습니다. Single LLM, RAG, Multi-Agent 구조를 비교한 뒤 각 Agent의 호출 로그와 위험도별 결과를 분석했습니다.

Single LLM→RAG→Multi-Agent→Stage-wise Evaluation

최종적으로 Response Agent가 생성한 응답을 Safety Agent가 검증하고, 문제가 있을 경우에만 Revision Agent를 호출하는 Conditional Bidirectional 구조를 설계했습니다.

05 · PROBLEM SOLVING

주요 문제와
해결

프로젝트 중간 점검에서 단순히 성능 수치만 제시해서는 최종 구조가 우수한 이유를 설명하기 어렵다는 피드백을 받았습니다. 이를 반영해 연구를 구조 비교 중심에서 Agent 기여도 분석 중심으로 확장했습니다.

분석 확장

  • Agent 호출 로그 분석
  • Revision 발생 원인 분석
  • Safety 피드백 반영 여부 확인

위험도별 검증

  • Low / Medium / High Risk 분리
  • 위험도별 상담 품질 변화 분석
  • 안전성과 공감 효과를 별도로 해석
06 · RESULT

결과와
검증

Response Agent 이후 Safety Agent가 검증하고 필요한 경우에만 Revision을 수행하는 Conditional Bidirectional 구조가 가장 우수한 성능을 보였습니다.

  • Safety Agent의 피드백이 실제 Revision에 반영됨을 로그 데이터로 검증했습니다.
  • 대부분의 수정이 위기 대응 문구와 전문가 상담 권고를 강화하는 방향으로 이루어졌습니다.
  • Medium Risk에서는 Safety와 Empathy가 함께 향상됐습니다.
  • High Risk에서는 Empathy 향상 효과가 특히 크게 나타났습니다.
07 · RETROSPECTIVE

회고와
개선 방향

높은 성능을 달성하는 것만큼 왜 해당 구조가 좋은지 설명할 수 있는 연구 설계가 중요하다는 점을 배웠습니다. 중간 피드백을 반영해 실험 방향을 수정하고, 구조의 성능뿐 아니라 Agent별 기여도를 분석하는 연구로 발전시킨 경험을 얻었습니다.

향후에는 더 다양한 LLM과 실제 사용자 평가를 추가해 일반화 성능을 검증하고, Retrieval 품질과 Agent 협업 전략을 함께 최적화하는 연구로 확장하고자 합니다.
